# Go Coding Standards

[Google Go Style Guide](https://google.github.io/styleguide/go/) に従う。

## Naming

- MixedCaps を使う。アンダースコアは使わない（テスト関数名・生成コード・OS互換を除く）
- パッケージ名は小文字の単一単語。`util`, `common`, `base`, `helper` は禁止
- ゲッターに `Get` プレフィックスは付けない。高コスト操作には `Compute` や `Fetch` を使う
- エクスポートする名前はパッケージ名の繰り返しを避ける（`http.Server` not `http.HTTPServer`）
- 定数は MixedCaps。`SCREAMING_SNAKE_CASE` や `K` プレフィックスは使わない
- レシーバ名は型名の1-2文字の略称。`self`, `this` は禁止。全メソッドで統一する
- 変数名の長さはスコープに比例させる。小スコープでは1文字（`r`, `w`, `i`）で可
- イニシャリズムは内部で一貫させる（`XMLAPI` or `xmlAPI`、`XMLApi` は不可）
- 単一型パッケージのコンストラクタは `New()` にする（`NewWidget()` ではなく）

## Imports

- グループ順: (1) 標準ライブラリ (2) サードパーティ (3) protobuf (4) 副作用import。空行で区切る
- ブランクimport（`import _`）は `main` パッケージとテストファイルのみ許可
- ドットimport（`import .`）は禁止
- import名の衝突時はプロジェクト固有側をリネームする（標準ライブラリ側ではない）

## Variable Declarations

- 非ゼロ値の初期化には `:=` を使う。意図的にゼロ値にする場合のみ `var x T` を使う
- チャネルは関数シグネチャで方向を指定する（`<-chan T`, `chan<- T`）
- スライス/マップの事前確保はサイズが確定しプロファイルで効果が出た場合のみ
- nil マップへの書き込みはパニックするため、書き込み前に初期化する
- スライスの空チェックは `len(s) == 0` を使う。`s == nil` ではない
- `sync.Mutex` を含む構造体はコピーしない。ポインタレシーバとポインタ渡しを使う

## Struct Literals

- 他パッケージの型は必ずフィールド名を使う
- ゼロ値フィールドは意味がある場合のみ明示する
- 複数行リテラルは閉じ括弧前にトレーリングカンマを付ける

## Error Handling

- エラーは必ずチェックする。`_` で握りつぶさない
- エラー変数は `Err` プレフィックス（`ErrNotFound`）
- エラーメッセージは小文字で始め、句読点で終わらない
- `%w` でラップ: 呼び出し側が `errors.Is()`/`errors.As()` で検査する場合。`%w` はフォーマット文字列の末尾に置く
- `%v` でラップ: システム境界（RPC, IPC, ストレージ）では独立したエラーにする
- エクスポート関数から具象エラー型を返さない。`error` インターフェースを返す
- 「何かが失敗した」だけの無情報なアノテーションは付けない
- ログして返すのではなく、上位に伝播させる。ログはトップレベルで取る
- エラー比較は `errors.Is()` を使う。文字列マッチは禁止
- 追加情報はカスタムエラー型のフィールドで構造的に表現する

## Panics and Recovery

- ライブラリで一時的なエラーに対して panic しない。error を返す
- 回復不能な不変条件違反には `log.Fatal` を使う（`panic` ではない）
- `Must` 関数はプログラム起動時またはテストでのみ使用可
- パッケージ境界を越えて panic を露出させない
- 予期しない panic をマスクするために `recover` を使わない

## Concurrency

- 非同期より同期関数を優先する。並行性は呼び出し側に任せる
- ゴルーチンのライフタイムを明示する。いつ終了するかドキュメントに書く
- キャンセルには `context.Context` を使う
- テスト内のゴルーチンから `t.Fatal` を呼ばない。`t.Error` + return を使う

## context.Context

- 常に第1引数（レシーバの次）。構造体フィールドに格納しない
- カスタムコンテキスト型を作らない
- オプション構造体に含めない

## Interfaces

- 具体的な必要性が生まれるまでインターフェースを定義しない
- 消費者側でインターフェースを定義する（提供者側ではない）
- インターフェースは小さく保つ
- 関数からは具象型を返す（実装を隠す必要がある場合を除く）

## Types and Receivers

- ポインタ vs 値レシーバ: 変更する・uncopyableフィールドがある・大きい型 → ポインタ。小さな値型 → 値
- 同一型のすべてのメソッドでレシーバの種類を統一する
- `interface{}` より `any` を使う
- 文字列出力には `%q` を使い空文字列を可視化する

## Style

- `gofmt` / `goimports` でフォーマットされた状態を保つ
- 早期リターンでネストを浅く保つ
- `init()` の使用は最小限にする
- `if` 文を途中で改行しない。複雑な条件は名前付き変数に抽出する
- `for` ループヘッダを途中で改行しない
- switch の `break` は不要（Go は自動で break する）
- 比較は変数を左に置く（`result == "foo"`）。Yoda スタイル禁止
- セキュリティ用途には `crypto/rand` を使う（`math/rand` ではない）

## Function Design

- 多数のパラメータにはオプション構造体またはバリアディックオプション関数を使う
- 関数宣言は1行に収める。収まらない場合はリファクタする

## Testing

- テストファイルは `_test.go` サフィックス
- テーブル駆動テストを優先する
- 失敗メッセージ: `YourFunc(%v) = %v, want %v` 形式。actual を先に出す（`got X, want Y`）
- 構造体比較は `cmp.Diff` / `cmp.Equal` を使う。`reflect.DeepEqual` は避ける
- 個々のフィールドではなく構造体全体を比較する
- サブテスト内では `t.Fatal`、サブテスト外では `t.Error` + `continue`
- テストヘルパーには `t.Helper()` を呼ぶ
- バリデーションヘルパーは `testing.T` を受けずに error を返す
- カスタムアサーションライブラリやテスト用DSLを作らない

## Documentation

- エクスポートされた名前にはドキュメントコメントを付ける
- コメントは対象の名前で始める（`// Server represents ...`）
- 非エクスポートでも非自明な挙動にはコメントを付ける
- パッケージコメントは `package` 句の直上に空行なしで配置する
- 自明なことをコメントしない（パラメータ型の再記述など）
- 非自明な条件分岐にはシグナルブーストコメントを付ける（例: `if err == nil { // if NO error`）
